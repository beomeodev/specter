# /ms.update-docs - Living Document Synchronization

Regenerate Living Documents based on recent code changes (TAG blocks + implementation).

## Overview

**Purpose**: Keep Living Documents synchronized with code after manual changes outside `/ms.implement`.

**What it updates**:
- `docs/api/[TAG-ID].md` - Living Documents (auto-generated from TAG blocks)

**What it does NOT update**:
- Constitution (manual updates only)
- spec.md, plan.md, tasks.md (updated by respective /ms.* commands)
- AGENTS.md (updated by `/ms.constitution`)

## Usage

```bash
/ms.update-docs                # All TAGs in current session
/ms.update-docs AUTH-001       # Specific TAG only
/ms.update-docs --all          # All TAGs in entire codebase
```

## Auto-Loaded Context

- `@.specify/memory/constitution.md` (Section IX - project constraints)
- `@AGENTS.md` (coding standards)

## Execution Steps

### Step 1: Determine Scope

**Parse argument**:

- **No argument**: Scan current session for changed files with TAG blocks
- **TAG-ID (e.g., AUTH-001)**: Update only this TAG
- **--all**: Scan entire codebase for all TAGs

**Output**:
```
Scope: [session / specific TAG / all TAGs]
```

### Step 2: Scan for TAG Blocks

**Use ripgrep to find TAG blocks**:

```bash
# Find all @CODE: tags
rg --no-heading --with-filename --line-number '@CODE:[A-Z0-9]+-\d{3}' src/ tests/
```

**Parse each TAG block** to extract:
- TAG ID (e.g., `@CODE:AUTH-001`)
- @SPEC path
- @TEST path
- @CREATED date
- @UPDATED date

**Filter by scope**:
- If specific TAG: Only process matching TAG
- If session: Only process TAGs in recently changed files
- If --all: Process all TAGs found

**Output**:
```
TAGs found: 12
TAGs to update: 3 (AUTH-001, USER-002, PAY-001)
```

### Step 3: Extract Information for Each TAG

For each TAG ID, gather information:

#### 3.1. SPEC Excerpt

**Read @SPEC path** (from TAG block):
```bash
# Example: specs/001-auth-spec/spec.md
```

**Extract FR section** matching TAG ID:
- Search for TAG ID in spec.md: `@SPEC:AUTH-001`
- Extract surrounding context (3-5 paragraphs)

**IF TAG not found in spec.md**:
- Note: "SPEC reference not found (implementation may have diverged)"

#### 3.2. Code Signatures

**Read source file** (file containing @CODE tag):

**TypeScript extraction** (regex-based, no AST):
```regex
export (function|const|class|interface|type) \w+[^{;]*
```

**Python extraction** (regex-based, no AST):
```regex
^def \w+\([^)]*\)( -> .+)?:|^class \w+(\([^)]*\))?:
```

**Extract**:
- Function signatures
- Class names
- Exported types/interfaces

**Example output**:
```typescript
export function authenticateUser(credentials: Credentials): Promise<AuthResult>
export interface AuthResult { token: string; user: User }
export class AuthService { ... }
```

#### 3.3. Test Examples

**Read test file** (from @TEST path in TAG block):

**Test extraction** (regex-based):
```regex
(it|test|describe)\(['"]([^'"]+)['"]
```

**Extract test descriptions**:
```
✓ should authenticate user with valid credentials
✓ should reject invalid credentials
✓ should handle expired tokens
```

### Step 4: Generate Living Document

**Template** (for each TAG):

```markdown
# Living Document: {TAG_ID}

*Auto-generated by `/ms.update-docs` on {CURRENT_DATE}*

## TAG Chain

{TAG_CHAIN}

**Status**: {STATUS}
**Created**: {CREATED_DATE}
**Last Updated**: {UPDATED_DATE}

## Specification Excerpt

**Source**: {SPEC_PATH}

{SPEC_EXCERPT}

## API Signatures

**Source**: {CODE_FILE_PATH}

```{LANGUAGE}
{FUNCTION_SIGNATURES}
```

## Test Coverage

**Source**: {TEST_PATH}

{TEST_EXAMPLES}

## Traceability

| Phase | Location | Status |
|-------|----------|--------|
| SPEC | {SPEC_PATH} | {SPEC_STATUS} |
| TEST | {TEST_PATH} | {TEST_STATUS} |
| CODE | {CODE_PATH} | ✅ |
| DOC | docs/api/{TAG_ID}.md | ✅ |

---

*This Living Document is auto-generated. Do not edit manually. Regenerate with `/ms.update-docs {TAG_ID}`*
```

**Placeholder replacement**:
- `{TAG_ID}` → AUTH-001
- `{CURRENT_DATE}` → 2025-10-16
- `{TAG_CHAIN}` → @SPEC:AUTH-001 → @TEST:AUTH-001 → @CODE:AUTH-001
- `{STATUS}` → implemented
- `{SPEC_EXCERPT}` → Extracted FR content
- `{FUNCTION_SIGNATURES}` → Extracted signatures
- `{TEST_EXAMPLES}` → Extracted test descriptions
- `{LANGUAGE}` → typescript / python
- `{SPEC_STATUS}`, `{TEST_STATUS}` → ✅ (if files exist) / ❌ (if missing)

### Step 5: Write Living Documents

For each TAG:

1. **Create directory** (if not exists):
   ```bash
   mkdir -p docs/api
   ```

2. **Write Living Document**:
   ```bash
   # Overwrite existing file
   write docs/api/{TAG_ID}.md
   ```

3. **Report**:
   ```
   ✅ Updated: docs/api/AUTH-001.md
   ```

## Error Handling

### Error 1: No TAGs Found

**Symptom**: ripgrep returns no @CODE: matches

**Message**:
```
⚠️ No TAG blocks found

Searched: src/, tests/
Pattern: @CODE:[A-Z0-9]+-\d{3}

Possible reasons:
1. No code has been implemented yet
2. TAG blocks not inserted (use `/ms.implement`)
3. Wrong directory structure

Run `/ms.implement` to generate code with TAG blocks.
```

**Exit**: Code 0 (not an error)

### Error 2: SPEC File Not Found

**Symptom**: @SPEC path in TAG block doesn't exist

**Action**: Continue with warning

**Message in Living Doc**:
```
## Specification Excerpt

**Source**: specs/001-auth-spec/spec.md

⚠️ SPEC file not found. TAG reference may be stale.
```

### Error 3: TEST File Not Found

**Symptom**: @TEST path in TAG block doesn't exist

**Action**: Continue with warning

**Message in Living Doc**:
```
## Test Coverage

**Source**: tests/auth/service.test.ts

⚠️ TEST file not found. Tests may need to be created.
```

### Error 4: Malformed TAG Block

**Symptom**: TAG block missing required fields (@SPEC, @TEST, @CHAIN)

**Message**:
```
❌ Error: Malformed TAG block in {FILE_PATH}:{LINE}

TAG ID: {TAG_ID}
Missing fields: @SPEC, @TEST

Please fix TAG block format. See AGENTS.md Section 4 for correct structure.
```

**Action**: Skip this TAG, continue with others

## Tools Used

- **Bash**: ripgrep for TAG scanning, mkdir for directory creation
- **Read**: Read SPEC files, TEST files, CODE files
- **Write**: Create Living Documents
- **Grep**: Extract function signatures, test descriptions (regex-based)
