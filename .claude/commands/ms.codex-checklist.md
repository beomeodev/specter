# /ms.codex-checklist - Independent PRD Checklist

Start a background Codex run that reads the source PRD set only and creates an
independent checklist for later PRD-to-Feature-Map verification.

This command is intentionally upstream of `/ms.verify`. It does not validate
`docs/prd/feature-map.md` directly and must not read it. Its job is to produce
an independent PRD commitment baseline before Claude compares that baseline to
the generated Feature Map.

## Usage

```bash
/ms.codex-checklist @docs/prd/PRD.md [@docs/prd/another.md]
/ms.codex-checklist --model gpt-5.6-luna --effort xhigh @docs/prd/PRD.md
```

Execution is always background. Users do not pass `--background`.

Default Codex runtime:

```text
model: gpt-5.6-luna
effort: xhigh
```

## Output

Codex must write:

```text
docs/prd/codex/checklist.md
```

## Execution Steps

### Step 0: Resolve Source PRDs

Use the PRD paths supplied in `$ARGUMENTS`.

If no PRD paths are supplied, read `docs/prd/feature-map.md` only to discover
the source PRD path list, not to inspect Feature content or ownership. If that
is not possible, stop and ask the user for the PRD paths.

Do not read:

- `docs/prd/feature-map.md` Feature sections or PRD Commitment Index
- `docs/prd/feature-map.checklist.md`
- `docs/prd/checklists/*`
- `specs/*`
- `plan.md`
- `tasks.md`

**Session read policy**: per AGENTS.md §2 — reuse files already read this session; a fresh `Read` immediately before `Edit`/`Write` is still required.

### Step 0.5: External Agent Preflight (session-level, once)

Apply the Preflight from `.claude/skills/specter-agent-protocols/SKILL.md` (§1). For this
command: a **single-agent station** — there is nothing to degrade to, so if Codex is still
unavailable after preflight + one retry, stop and report the failure; do not run `/ms.verify`
against a missing independent baseline.

### Step 1: Start Codex In Background

Invoke the Codex plugin rescue command in background mode with default runtime
selection unless the user explicitly provided overrides:

```text
/codex:rescue --background --fresh --model gpt-5.6-luna --effort xhigh <prompt>
```

If the user provided `--model` or `--effort`, pass those values through instead
of the defaults.

### Step 2: Codex Prompt

Forward this task to Codex:

```text
You are creating an independent PRD-to-FeatureMap audit checklist for SPECTER.

Read ONLY the source PRD documents listed below.
Do NOT read docs/prd/feature-map.md, specs/, plan.md, tasks.md, or existing
checklist files. Your job is not to create the Feature Map. Your job is to
create the checklist that will later verify whether a Feature Map preserved the
PRD.

Source PRDs:
<insert PRD path list>

Extract every durable PRD commitment:
- functional requirements
- user journeys
- acceptance criteria
- constraints
- non-functional requirements
- integration promises
- data, storage, migration, privacy, logging, audit, and security promises
- explicit exclusions or deferred work
- product principles that constrain behavior

For each commitment, assign a stable ID C001, C002, ...
Include the PRD reference, commitment type, concise requirement summary, and how
it must appear in a Feature Map.

Rules:
- Do not invent Features.
- Do not decide final Feature ownership.
- Do not merge separate commitments just because they sound related.
- Do not drop small, operational, cross-cutting, or negative requirements.
- If the PRD is ambiguous, record an Ambiguity item instead of resolving it silently.
- Write persisted output in English.
- Keep the output usable by another agent comparing it to docs/prd/feature-map.md.

Write the checklist to docs/prd/codex/checklist.md with this structure:

# Codex PRD-to-FeatureMap Checklist

**Mode**: prd-only
**Generated By**: Codex
**Source PRDs**:
- <path>

## Commitment Inventory

| ID | PRD Ref | Type | Requirement / Commitment | Must Appear In Feature Map As |
| --- | --- | --- | --- | --- |

## Acceptance Criteria Inventory

| ID | PRD Ref | Acceptance Criterion | Expected Feature Map Handling |
| --- | --- | --- | --- |

## Cross-Cutting Concerns

| ID | PRD Ref | Concern | Expected Ownership Rule |
| --- | --- | --- | --- |

## Data / Migration / Integration Promises

| ID | PRD Ref | Promise | Expected Feature Map Handling |
| --- | --- | --- | --- |

## Explicit Exclusions / Deferred Work

| ID | PRD Ref | Exclusion Or Deferred Work | Expected Feature Map Handling |
| --- | --- | --- | --- |

## Ambiguities

| ID | PRD Ref | Ambiguity | Why It Matters For Feature Mapping |
| --- | --- | --- | --- |

## Slicing Risks

| Risk | Why It Matters | What To Check In Feature Map |
| --- | --- | --- |

## Verification Checklist

- [ ] Every commitment above has exactly one owning Feature.
- [ ] Every acceptance criterion is represented in a Feature done criterion or test expectation.
- [ ] Every explicit exclusion has a destination Feature.
- [ ] Every cross-cutting concern has one real owner, with earlier stubs only if needed.
- [ ] Every data, migration, and integration promise is assigned to a Feature.
- [ ] No user-facing behavior is diluted into vague implementation wording.
```

Also echo the finished checklist between ===REPORT BEGIN=== and ===REPORT END=== markers in your
final message, verbatim, so it can be salvaged if the file write fails.

**Report-Write Protocol**: apply `specter-agent-protocols` §3 to `docs/prd/codex/checklist.md`
(expected marker line: `**Mode**: prd-only`). The consuming command (`/ms.verify`,
`/ms.pre-specter`) runs the deterministic check; on a missing/partial file after one retry,
salvage from the `===REPORT BEGIN===`/`===REPORT END===` markers instead of re-running the whole
background job blind or stopping outright.

### Step 2.5: Run-State Ledger (bookkeeping, not a gate)

Append one line to `.specify/specter-run.jsonl` (create it if needed; append-only, never
rewritten — a missing/corrupt ledger never blocks this command). Since this step is a background
kickoff, not a finished verdict, record `PENDING`; the consuming command (`/ms.verify` or
`/ms.pre-specter`) already checks the artifact directly and does not depend on this entry being
upgraded later:

```bash
mkdir -p .specify
printf '{"ts":"%s","cycle":"pre","feature":null,"step":"codex-checklist","verdict":"PENDING","artifacts":["docs/prd/codex/checklist.md"]}\n' \
  "$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> .specify/specter-run.jsonl
```

### Step 3: Report

Display in Korean:

```text
Codex PRD checklist 작업을 백그라운드에서 시작했습니다.

📄 예상 산출물: docs/prd/codex/checklist.md
🎯 다음 단계: /ms.verify

/ms.verify는 Codex 결과가 아직 없으면 대기 안내를 표시하고 중단합니다.
```
