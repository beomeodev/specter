## 기능

TRUST 5 원칙 적용 및 헌법 준수 확인을 통한 자동 코드 검토.

**주요 기능**:
- ✅ TRUST 5 원칙 검증 (테스트 우선, 가독성, 통일성, 보안성, 추적성)
- ✅ 헌법 준수 확인 (파일 크기, 함수 복잡성, 보안 규칙)
- ✅ 코드 품질 메트릭 (테스트 커버리지, 린팅, 타입 체킹)
- ✅ 보안 취약점 스캐닝
- ✅ 성능 모범 사례
- ✅ 문서 완전성 확인
- ✅ TAG 체인 무결성 검증

## 사용 시기

**자동 트리거**:
- `/fin` 워크플로 중 (커밋 전 품질 게이트)
- 코드 검토 요청
- SPEC 구현 완료 (`/ms.implement` 후 단계)
- 풀 리퀘스트 생성

**수동 호출**:
- TRUST 5 준수를 위한 코드 검토
- 헌법 준수 확인
- 보안 모범 사례 검증
- 커밋 전 코드 품질 평가
- 병합 전 품질 게이트

**일반적인 시나리오**:
1. **커밋 전**: 코드가 품질 표준을 충족하는지 확인
2. **구현 후**: TRUST 5 준수 확인
3. **코드 검토 중**: 체계적인 피드백 제공
4. **배포 전**: 최종 품질 확인

## 작동 방식

### 검토 체크리스트 (TRUST 5 프레임워크)

#### 1. **T**est-First (테스트 커버리지 및 품질)

**확인**:
- [ ] 테스트 커버리지 ≥85% (헌법 섹션 I)
- [ ] 모든 중요한 경로에 대한 테스트 존재 (인증, 결제, 보안)
- [ ] 테스트가 RED → GREEN → REFACTOR 주기를 따름
- [ ] 테스트 이름이 동작을 명확하게 설명
- [ ] 주석 처리된 테스트 없음
- [ ] 테스트 격리 (공유 상태 없음)

**명령**:
```bash
# 파이썬 테스트 커버리지
pytest tests/ --cov=src/ --cov-report=term-missing --cov-report=html
# ≥85% 커버리지를 보여야 함

# 타입스크립트 테스트 커버리지
npm test -- --coverage
# ≥85% 커버리지를 보여야 함
```

**실패 예시**:
- ❌ 85% 미만 커버리지: "커버되지 않은 코드 경로에 대한 테스트 추가"
- ❌ 새 기능에 대한 테스트 없음: "구현 전 테스트 작성 (TDD RED 단계)"
- ❌ 테스트에 부작용이 있음: "테스트 격리 보장, 상태 정리"

---

#### 2. **R**eadable (코드 명확성 및 단순성)

**확인**:
- [ ] 파일 ≤500 SLOC (주석 제외 소스 코드 라인)
- [ ] 함수 ≤100 LOC (코드 라인)
- [ ] 함수 매개변수 ≤5
- [ ] 중첩 깊이 ≤4 수준
- [ ] 함수당 순환 복잡도 ≤10
- [ ] 명확한 변수 이름 (표준이 아닌 약어 없음)
- [ ] 함수는 단일 책임을 가짐

**명령**:
```bash
# 파이썬 복잡성 확인
ruff check src/ --select C90 --config "C90 = {max-complexity = 10}"

# 타입스크립트 복잡성 확인
npx eslint src/ --rule 'complexity: ["error", 10]'

# 파일 라인 수 계산 (SLOC)
python -c "
import sys
with open(sys.argv[1]) as f:
    lines = [l for l in f if l.strip() and not l.strip().startswith('#')]
    print(f'{len(lines)} SLOC')
" <file_path>
```

**실패 예시**:
- ❌ 파일이 500 SLOC 초과: "파일을 더 작은 모듈로 분할"
- ❌ 함수가 100 LOC 초과: "헬퍼 함수 추출"
- ❌ 복잡도 > 10: "조건부 로직 단순화, 분기 축소"
- ❌ 잘못된 이름 지정: "`d`를 `database_connection`으로 이름 변경"

---

#### 3. **U**nified (타입 안전성 및 일관성)

**확인**:
- [ ] 타입스크립트 `strict: true` 또는 파이썬 `mypy --strict` 통과
- [ ] 타입 오류 없음
- [ ] 일관된 코드 스타일 (린터 통과)
- [ ] 타입스크립트에 `any` 타입 없음 (적절한 타이핑 사용)
- [ ] 타입 없는 파이썬 함수 없음 (타입 힌트 추가)
- [ ] TAG 블록 존재 및 정확

**명령**:
```bash
# 타입스크립트 타입 확인
npx tsc --noEmit
# 종료 코드 0으로 종료해야 함

# 파이썬 타입 확인
mypy src/ --strict
# 종료 코드 0으로 종료해야 함

# 린팅
ruff check src/      # 파이썬
npx eslint src/      # 타입스크립트
```

**실패 예시**:
- ❌ 타입 오류: "타입 불일치 수정, 적절한 타입 추가"
- ❌ `any` 남용: "`any`를 특정 타입으로 교체"
- ❌ 린터 경고: "커밋 전 모든 린팅 문제 수정"
- ❌ TAG 블록 누락: "새 파일에 @CODE TAG 블록 추가"

---

#### 4. **S**ecured (보안 모범 사례)

**확인**:
- [ ] 하드코딩된 비밀 없음 (비밀번호, API 키, 토큰)
- [ ] 모든 사용자 입력에 대한 입력 유효성 검사
- [ ] SQL 매개변수화 (문자열 보간 없음)
- [ ] XSS 방지 (HTML 이스케이프, CSP 헤더)
- [ ] CSRF 보호 (토큰, SameSite 쿠키)
- [ ] 비밀번호 해싱 (bcrypt/argon2, ≥10 라운드)
- [ ] 모든 외부 통신에 HTTPS 사용
- [ ] 사용자 입력으로 `eval()` 또는 `exec()` 사용 안 함

**명령**:
```bash
# 하드코딩된 비밀 확인
rg -i 'password\s*=\s*["\']' src/
rg -i 'api[_-]?key\s*=\s*["\']' src/
# 결과가 없어야 함

# 파이썬 보안 스캔
pip install bandit
bandit -r src/ -f json

# 타입스크립트/자바스크립트 보안 스캔
npm audit
# 0개의 취약점을 보여야 함
```

**실패 예시**:
- ❌ 하드코딩된 비밀번호: "환경 변수로 이동"
- ❌ SQL 주입 위험: "매개변수화된 쿼리 사용"
- ❌ 입력 유효성 검사 없음: "모든 사용자 입력 유효성 검사"
- ❌ 약한 해싱: "≥10 라운드의 bcrypt 사용"

---

#### 5. **T**rackable (추적성 및 문서화)

**확인**:
- [ ] TAG 블록 존재 (@CODE, @SPEC, @TEST, @CHAIN)
- [ ] TAG 체인 완료 (SPEC → TEST → CODE)
- [ ] 고아 TAG 없음 (파일 없는 TAG)
- [ ] 공개 API가 변경된 경우 README 업데이트됨
- [ ] 사용자 대상 변경 사항으로 CHANGELOG 업데이트됨
- [ ] 주석은 무엇이 아닌 왜를 설명
- [ ] API 문서 존재 (독스트링)

**명령**:
```bash
# TAG 블록 스캔
rg '@(SPEC|TEST|CODE|DOC):' -n --type-add 'md:*.md' --type md --type py --type ts

# TAG 무결성 확인
python .claude/hooks/ms/core/project.py
# 높은 무결성 백분율을 보여야 함

# 고아 TAG 찾기
# 예: @CODE:AUTH-001이 있지만 파일을 찾을 수 없음
rg '@CODE:AUTH-001' -l
# 하나 이상의 파일을 반환해야 함
```

**실패 예시**:
- ❌ TAG 블록 누락: "새 파일에 @CODE TAG 블록 추가"
- ❌ 끊어진 TAG 체인: "@SPEC → @TEST → @CODE 체인 완료 확인"
- ❌ 고아 TAG: "TAG 제거 또는 누락된 파일 복원"
- ❌ 독스트링 없음: "공개 함수에 독스트링 추가"

## 실패 모드

### 코드 검토가 실패하거나 차단될 때:

1. **도구 미설치**: 린터, 타입 체커 또는 테스트 러너 누락
   - **해결책**: 언어별 필수 도구 설치 (헌법 참조)

2. **구성 누락**: `tsconfig.json`, `pyproject.toml` 등 없음
   - **해결책**: 구성 파일 초기화

3. **테스트 시간 초과**: 테스트 스위트 실행에 몇 분 소요
   - **해결책**: 테스트 병렬화, 외부 종속성 모의

4. **타입 오류 너무 많음**: 수정해야 할 수백 개의 타입 오류
   - **해결책**: 점진적으로 타입 힌트 추가, TODO와 함께 임시로 `# type: ignore` 사용

5. **보안 스캔 오탐**: 도구가 문제가 아닌 것을 보고함
   - **해결책**: 수동 검토, 정당화와 함께 억제 주석 추가

## 모범 사례

### ✅ 해야 할 일:

1. **순서대로 검토**: TRUST 5 순서 따르기
   - 테스트로 시작 (커버리지 확인)
   - 그 다음 가독성 (복잡성)
   - 그 다음 타입 (타입 안전성)
   - 그 다음 보안 (취약점)
   - 마지막으로 추적성 (TAG 체인)

2. **가능한 한 자동화**:
   - 린터, 타입 체커, 보안 스캐너 사용
   - CI/CD에서 자동으로 테스트 실행
   - 사전 커밋 훅에서 TAG 무결성 검증

3. **문제 명확하게 문서화**:
   - 파일 경로 및 줄 번호 포함
   - 문제인 이유 설명
   - 구체적인 수정 제안

4. **헌법 준수 확인**:
   - 파일 크기 ≤500 SLOC
   - 함수 복잡성 ≤10
   - 테스트 커버리지 ≥85%

5. **영향이 큰 문제에 먼저 집중**:
   - 보안 취약점 (P0)
   - 실패하는 테스트 (P0)
   - 누락된 테스트 (P1)
   - 코드 품질 (P2)
   - 스타일 문제 (P3)

### ❌ 하지 말아야 할 일:

1. **스타일 문제에 집착하지 않기**
   - 린터가 처리하도록 둠
   - 로직과 디자인에 집중

2. **사소한 문제로 차단하지 않기**
   - 보안/정확성: 차단
   - 스타일/이름 지정: 차단하지 않는 제안

3. **보안 확인 건너뛰지 않기**
   - 항상 보안 스캐너 실행
   - 항상 하드코딩된 비밀 확인

4. **컨텍스트 없이 검토하지 않기**
   - 먼저 SPEC 읽기
   - 기능 요구 사항 이해
   - 관련 파일 확인

5. **테스트 실행 없이 승인하지 않기**
   - 전체 테스트 스위트 실행
   - 테스트 커버리지 보고서 확인
   - 테스트가 실제로 동작을 테스트하는지 확인

## 예시

### 예시 1: 전체 검토 출력

**입력**: `src/auth/service.py` 검토

**출력**:
```markdown
## 코드 검토: src/auth/service.py

### ✅ 통과: 테스트 우선
- 테스트 커버리지: 92% (85% 요구 사항 초과)
- 모든 중요한 경로 테스트됨
- 테스트가 TDD 워크플로를 따름

### ❌ 실패: 가독성
- `validate_and_authenticate` 함수가 127 LOC (100 LOC 제한 초과)
  - **수정**: `validate_credentials()` 헬퍼 함수 추출
- `handle_login_flow`의 복잡성: 14 (10 초과)
  - **수정**: 조건부 로직 단순화, 중첩된 if 문 축소

### ✅ 통과: 통일성
- 타입 확인: 오류 0개
- 모든 함수에 타입 힌트 있음
- 린팅: 경고 0개

### ⚠️ 경고: 보안
- 45행: `email`에 악의적인 입력이 포함된 경우 잠재적인 SQL 주입
  - **수정**: 매개변수화된 쿼리 사용: `User.query.filter_by(email=email).first()`
- 로그인 엔드포인트에 속도 제한 없음
  - **권장 사항**: 무차별 대입 공격을 방지하기 위해 속도 제한 추가

### ✅ 통과: 추적성
- TAG 블록 존재: @CODE:AUTH-001
- TAG 체인 완료: @SPEC:AUTH-001 → @TEST:AUTH-001 → @CODE:AUTH-001
- 독스트링 존재

### 요약
**차단 문제**: 1개의 보안 문제 (SQL 주입 위험)
**비차단 문제**: 2개의 가독성 문제 (함수가 너무 길고 복잡성이 너무 높음)

**필요한 조치**: 병합 전 SQL 주입 수정
```

### 예시 2: 빠른 품질 확인

**입력**: 코드가 커밋 준비되었는지 확인

**출력**:
```markdown
## 품질 게이트: 차단됨 ❌

**차단 문제**:
1. 테스트 커버리지: 62% (85% 요구 사항 미만)

2. 타입 오류: `src/payment/types.ts`에서 3개의 오류
   - 수정: `processPayment()` 함수에 적절한 타입 추가
3. 보안: 1개의 높은 심각도의 npm 취약점
   - 실행: `npm audit fix`

**비차단 문제**:
1. 파일 `src/utils/helpers.py`가 543 SLOC (500 제한 초과)
   - 고려 사항: 더 작은 모듈로 분할

**권장 사항**: 커밋 전 차단 문제 수정
```

## 참조

**헌법**:
- 섹션 I: 테스트 우선 개발 (≥85% 커버리지)
- 섹션 II: 단순성 우선 아키텍처 (≤500 SLOC 파일, ≤100 LOC 함수, ≤10 복잡성)
- 섹션 V: TRUST 5 원칙 (포괄적인 품질 프레임워크)
- 섹션 VII: 기본 보안 (보안 요구 사항)

**스킬**:
- `ms-foundation-trust`: TRUST 5 유효성 검사 프레임워크
- `ms-foundation-constitution`: 헌법 원칙 참조
- `ms-lang-python`: 파이썬 코드 품질 도구
- `ms-lang-typescript`: 타입스크립트 코드 품질 도구

**외부 리소스**:
- 테스트 커버리지 도구: `pytest --cov`, `jest --coverage`
